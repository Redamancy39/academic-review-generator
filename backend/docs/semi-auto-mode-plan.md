# 半自动模式实施规划

## 一、需求分析

### 1.1 全自动模式（当前）
- 用户提交主题后，系统自动完成所有阶段
- 用户只能观察进度，无法干预
- 适用于：用户信任系统、时间紧迫、批量处理

### 1.2 半自动模式（新增）
- 指定阶段暂停，等待用户确认
- 用户可查看中间结果、提出修改意见
- 系统根据用户反馈调整后续流程
- 适用于：高质量要求、复杂主题、关键应用

### 1.3 可配置的暂停点

| 暂停点 | 用户可审核内容 | 用户可修改内容 |
|--------|---------------|---------------|
| `after_planning` | 检索策略、章节结构 | 调整检索词、修改章节 |
| `after_screening` | 筛选的文献列表 | 增删文献、调整权重 |
| `after_writing` | 初稿内容 | 修改段落、调整结构 |
| `after_review` | 审稿意见和修订稿 | 确认是否接受、指定修改方向 |

## 二、架构设计

### 2.1 数据模型

```python
# RunConfig 新增字段
class RunConfig:
    # ... 现有字段 ...
    mode: str = "auto"  # "auto" | "semi-auto"
    pause_points: List[str] = []  # ["after_planning", "after_screening", ...]

# 任务状态新增
class TaskStatus:
    # ... 现有字段 ...
    pause_reason: Optional[str] = None  # "after_planning" 等
    user_feedback: Optional[Dict] = None  # 用户的审核反馈
```

### 2.2 API 设计

```python
# 新增 API 端点

# 1. 确认当前阶段，继续执行
POST /api/v1/reviews/{task_id}/confirm
{
    "action": "continue",  # 或 "revise"
    "modifications": {     # 可选的用户修改
        "keywords": ["新增关键词"],
        "remove_paper_ids": ["REF001"],
        "add_paper_ids": ["REF050"],
        "section_comments": {
            "3.1": "需要增加对比表格"
        }
    }
}

# 2. 请求人工修改（重新执行当前阶段）
POST /api/v1/reviews/{task_id}/revise
{
    "instructions": "请重新筛选，重点关注 XX 方向的文献"
}
```

### 2.3 工作流调整

```python
async def run(self, ...):
    for stage in stages:
        result = await self._execute_stage(stage)

        # 检查是否需要在当前阶段后暂停
        if self._should_pause(stage):
            self._set_pause_state(stage)
            return {
                "status": "paused",
                "pause_reason": stage,
                "stage_result": result,
                "awaiting_user_action": True,
            }

        # 处理用户反馈（如果有）
        if self.user_feedback:
            await self._apply_user_feedback(stage, self.user_feedback)
```

## 六、风险和缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 用户长时间不确认 | 任务占用资源 | 设置超时自动继续或终止 |
| 用户修改导致数据不一致 | 后续阶段出错 | 验证用户输入，提供回滚 |
| 前端刷新丢失状态 | 用户操作丢失 | 状态持久化到后端 |
